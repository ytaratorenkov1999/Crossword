# Инструкция: как правильно составлять кроссворды

---

## Структура одного кроссворда

Каждый кроссворд в файле `questions.js` — это объект со следующими полями:

```js
{ id: 1, title: 'Кроссворд #1', difficulty: 1,
  grid: [ ... ],
  words: [ ... ]
}
```

| Поле | Что это |
|---|---|
| `id` | Порядковый номер (1, 2, 3...) |
| `title` | Название, отображается при открытии кроссворда |
| `difficulty` | Сложность: `1` = лёгкий, `2` = средний, `3` = сложный |
| `grid` | Двумерный массив — сетка клеток |
| `words` | Массив слов с подсказками и координатами |

---

## Шаг 1 — Нарисуй сетку на бумаге

Прежде чем писать код — нарисуй сетку от руки на клетчатой бумаге.

- Обозначь чёрные клетки (блокираторы)
- Пронумеруй начало каждого слова (цифра в углу клетки)
- Определи направление каждого слова: **→ горизонталь** (across) или **↓ вертикаль** (down)

**Правило нумерации:** номер ставится в первую клетку слова. Если в одной клетке начинаются и горизонталь, и вертикаль — они получают **один и тот же номер**.

---

## Шаг 2 — Запиши сетку в grid

Сетка — это массив строк, каждая строка — массив клеток.

| Значение клетки | Что означает |
|---|---|
| `'.'` | Чёрная (заблокированная) клетка |
| `' '` | Белая клетка без номера |
| `'1'`, `'2'`... | Белая клетка, с которой начинается слово №N |

**Пример сетки 6×6:**

```
Бумага:          Код:
. 1 _ _ _ .      ['.','1',' ',' ',' ','.'],
_ _ . 2 _ _      [' ',' ','.','2',' ',' '],
_ _ _ _ _ _      [' ',' ',' ',' ',' ',' '],
```

> Строки в grid — это строки сетки сверху вниз (row 0, row 1...).  
> Столбцы — слева направо (col 0, col 1...).

---

## Шаг 3 — Подбери слова и посчитай координаты

Для каждого слова нужно точно знать:

- **row** — номер строки, где начинается слово (считай с 0 сверху)
- **col** — номер столбца, где начинается слово (считай с 0 слева)
- **length** — количество букв (считай клетки до следующей чёрной или края)
- **answer** — само слово заглавными буквами, ровно `length` букв

```js
{ number: 1, direction: 'across', row: 0, col: 1, length: 4,
  clue: 'Капли воды на траве по утрам', answer: 'РОСА' }
```

> ⚠️ **Главная ошибка** — когда `length` не совпадает с реальной длиной `answer`.  
> Всегда проверяй: `answer.length === length`.

---

## Шаг 4 — Проверь пересечения

Если горизонтальное и вертикальное слово пересекаются в одной клетке — буква в этой клетке должна совпадать у обоих слов.

**Как проверить вручную:**

1. Найди клетку пересечения (row, col)
2. Определи, какая это буква в горизонтальном слове: позиция = `col - word.col`
3. Определи, какая это буква в вертикальном слове: позиция = `row - word.row`
4. Убедись, что буквы одинаковые

**Пример:**

```
Горизонталь: РОСА  — начало row:0, col:1
Вертикаль:   БОР   — начало row:0, col:2

Пересечение: row:0, col:2
  РОСА[col:2 - col:1 = позиция 1] = 'О'
  БОР [row:0 - row:0 = позиция 0] = 'Б'
  ❌ Не совпадают — нужно менять слово!
```

---

## Шаг 5 — Напиши объект words

Пример полного корректного слова:

```js
{ number: 3,            // номер в сетке
  direction: 'across',  // 'across' = горизонталь, 'down' = вертикаль
  row: 1,               // строка начала (0 = верхняя)
  col: 3,               // столбец начала (0 = левый)
  length: 3,            // количество клеток
  clue: 'Домашнее мурлыкающее животное',  // подсказка
  answer: 'КОТ'         // ответ заглавными, ровно length букв
}
```

---

## Чеклист перед сохранением

Перед тем как вставить кроссворд в файл — пройдись по списку:

- [ ] Каждая клетка `'N'` в grid соответствует началу слова с `number: N` в words
- [ ] `answer.length === length` для каждого слова
- [ ] Все пересечения (горизонталь × вертикаль) совпадают по букве
- [ ] Слова не выходят за пределы сетки (`col + length <= ширина сетки`)
- [ ] Нет слов длиной 1 клетка
- [ ] В `answer` только заглавные русские буквы, без пробелов

---

## Типичные ошибки

| Ошибка | Что происходит |
|---|---|
| `answer: 'ЛИМО'` при `length: 4` (слово «лимон» = 5 букв) | Слово никогда не засчитается |
| Пересечение: буквы не совпадают | Игра не проверяет это — но буква будет «сломана» |
| `row` или `col` указан неверно | Слово рисуется не там |
| `length` больше, чем белых клеток | Слово выходит за границу |
| Пустые клетки между словом и чёрной клеткой не учтены | Реальная длина слова меньше, чем кажется |

## Быстрый шаблон для нового кроссворда

```js
{ id: 11, title: 'Кроссворд #11', difficulty: 1,
  grid: [
    ['1',' ',' ','.'],
    [' ','.','2',' '],
    [' ',' ',' ',' '],
    ['.','3',' ',' '],
  ],
  words: [
    { number:1, direction:'across', row:0, col:0, length:3,
      clue:'Подсказка', answer:'СОМ' },
    { number:2, direction:'across', row:1, col:2, length:2,
      clue:'Подсказка', answer:'ОН' },
    { number:3, direction:'across', row:3, col:1, length:3,
      clue:'Подсказка', answer:'БОР' },
  ]
},
```

---

## Совет

Самый надёжный способ — нарисовать сетку на листе бумаги в клетку, написать слова от руки, проверить все пересечения глазами, и только потом переносить в код. Так ошибок почти не бывает.
